到底要不要学习命令

办公和家用完全不需要学习命令：

  1. 如果仅仅是使用Linux系统，比如办公，不涉及系统安装/维护/配置，那么完全没必要学习命令，浪费时间不说，还用不上。

  2. 如果想自己对自己的电脑做一些简单的维护，包括系统安装、软件安装、配置等，那么可以了解一些简单的命令，平时能用到的不超过20条；并且，仅需了解，不需要学习，更不需要完全掌握，用到的时候现抄完全来得及。还有一点，就是学习命令最主要的目的是提高效率，让电脑操作更流畅和更快捷，而并不是掌握某种特别的技能。尤其效率这方面，与图形界面相比，命令有显著优势。

  3. 如果想在与Linux相关的行业发展，比如系统运维、开发等，那么肯定要学习命令，因为很多服务需要通过命令来实现。

驿窗注：windows系统运维需要使用命令

事实上，如果是从事windows系统的运维工作，也需要掌握大量的命令才行。完全不了解命令是无法完成运维工作的。从这个角度看，Linux与windows没有区别。

之所以很多人误以为想用Linux就必须学习命令，是因为国内现有的Linux经典书籍，比如《鸟哥的Linux私房菜》，里面大量的命令讲解，国内有很多人都读过甚至精读过这本书。还有就是国内写Linux书籍的人，基本都是从事运维或者开发相关工作的。在他们的实际工作中，确实是离不开命令，所以书里不可避免要涉及大量与命令相关的内容。国内很少有Linux书籍是通过非命令行的方式来讨论Linux系统操作的。

所以，基本上，能找到的Linux书籍都是写给运维人员看的，而不是写给办公室操作人员看的。

看面向运维的书来学习如何使用Linux，有点像是用大炮打蚊子，这不对。

所以，如果您并不是从事运维或开发工作，那么相信我，不用学命令，也不要去学命令。

上面第2条维护自己的电脑，这种情况下必然涉及系统安装、维护、配置等操作，如果使用图形界面来完成会比较繁琐且浪费时间精力，改用命令的话一方面效率非常高，另一方面涉及的命令也很少很简单，所以完全不需要去深入学习命令本身，只需要知道可以用命令来实现即可，真到了要使用命令的时候，现到网上查都来得及，甚至复制过来回车即可，就这么简单！

比如自己维护电脑可能涉及的命令：

  1. sudo cp sources.list /etc/apt/sources.list

  2. sudo apt update

  3. sudo apt upgrade

  4. sudo apt install gufw

(Linux命令严格区分大小写)
