用live-iso映像体验Linux系统

危险

驿窗提示：提前备份数据；未成年人请征得父母或监护人同意后再进行操作

虽然体验过程不会破坏电脑硬盘中的数据，但体验系统中确实有工具能够修改硬盘，比如硬盘分区和格式化，这会导致电脑中已有数据丢失。所以，为保险起见，建议一定要提前备份重要数据；如果操作者是未成年人，务必要征得父母或监护人同意后再进行操作，以确保数据安全。

本节内容假设您已经有一台没有重要数据的windows电脑。

驿窗注：本节内容有视频版本

本节内容的视频版本在B站《Linux入门指南》P3. 创建体验用的启动U盘 。

如果您平时只使用windows系统，对Linux系统并不了解，那么最简单的方法就是先体验一下Linux系统，找找感觉。

体验的过程是先自己制作一个Linux启动U盘，用这个U盘来引导您的Windows电脑，然后在引导起来的Linux系统中操作，看看Linux系统与windows有什么不同。U盘启动电脑以后会是一个完整的Linux系统，是图形界面，您可以尝试几乎所有的Linux系统功能，比如尝试看网络视频、收发邮件、安装Linux软件等。整个体验过程不会损坏原有的windows系统，随时可以重启进入原有的windows系统进行其它工作。

驿窗注：系统时间可能会被修改

体验过程可能会导致您原windows系统的时间被修改。体验完再启动电脑进入windows系统后如果发现时间不对，直接改回正确的时间即可。

体验可能需要您动手做一些准备工作，请参考下面内容：

  1. 备份重要数据；

  2. 准备一个容量不小于8G的U盘；

  3. 下载Debian Live iso映像文件并验证；

  4. 下载U盘创建工具：01iso_xp-04win32-imager；

  5. 创建启动U盘；

  6. 用启动U盘引导您的windows电脑；

  7. 体验Debian，比如上网看视频、收发邮件等等；

  8. 体验完以后，在Linux系统中关闭电脑；

  9. 重新启动windows系统；

  10. 检查时间是否正确，查看重要数据是否损坏；

  11. 结束。

1. 备份重要数据

无论电脑中有几块硬盘，都建议把重要数据备份到移动硬盘上。最佳方案是在多个硬盘上建立多个备份，这样数据安全更有保障。

用Debian Live iso引导电脑后在体验的过程中，主要是体验Linux系统与windows系统的差别，并不做对硬盘有修改的操作，不会对硬盘上的数据造成损坏。所以，这里的备份只是为了以防万一。毕竟，数据最重要。

不过，因为时差问题，体验过程中可能会导致电脑系统的时间被修改，这个修改的是bios时间，windows系统本身并不会被破坏，所以，如果重新启动到windows后发现时间不对，改回即可。时间改回操作可以在bios中进行，也可以在windows系统中进行。

2. 准备一个容量不小于8G的U盘

实际上，创建Debian Live启动U盘只需要大概4G的存储空间就够了，但为了防止意外问题，驿窗建议用8G或更大U盘。

在创建启动U盘时，U盘中的所有数据都会丢失。所以，如果U盘中有重要数据，请提前备份到其它位置。

3. 下载Debian Live iso映像文件并校验哈希值

3.1. 下载Debian Live iso映像文件

Debian Live iso映像有不同的版本，常用的是KDE、Gnome、XFCE这三个版本，下载页面：Debian nonfree目录 。

这个目录里有很多文件，我们只需要下载iso文件就可以。目录中有多个不同的iso文件，区别就是桌面环境不同。与windows只有一个桌面风格不同，Linux系统可以使用多种不同的桌面环境。比如，如果您下载的iso文件名带有gnome字样，表示桌面环境是gnome桌面；如果文件名带有kde字样，则表示桌面环境是kde plasma环境，依此类推。各种桌面环境的区别请参考本书附录。

如果您并不熟悉Linux系统，建议您考虑gnome和kde这两个桌面环境，因为这两个是使用人数相对比较多的桌面。Gnome的风格与手机比较像，kde的风格与windows比较像。因为风格的差异，导致二者在操作上也有很大不同。

上面提供的目录位于Debian官方服务器，您也可以从国内的镜像服务器下载，比如中国科大镜像服务器 。

3.2. 校验Debian Live iso映像文件的哈希值

把iso映像文件下载到您的电脑中以后，请一定要校验下载文件的哈希值。校验的意义是确保您下载到本地的文件与服务器上提供的文件完全一致，这表示文件在下载过程中没有任何损坏。

在下载页面的iso映像文件目录中，同时会有名为SHA256SUMS的文件，这个文件里保存的是所有iso文件的SHA256哈希值。您把iso映像文件下载到本地以后，用SHA256哈希校验工具对iso映像文件进行校验，得到的sha256哈希值如果与目录中为SHA256SUMS文件里提供的哈希值完全一致，那么就表示下载到本地的iso映像文件是没有问题的。如果不完全一致，那么这个文件就已经损坏，不能使用。这一点很重要！

哈希值校验方法：

下载一个哈希校验工具，如果没有的话，可以下载7-Zip，它可以进行哈希校验。下载7-Zip并安装完成后，找到下载的iso映像文件，用鼠标右键点击iso映像文件，选择CRC SHA > SHA-256，之后稍微等待一下，7-Zip会计算出此文件的SHA256哈希值，对比一下这个值与SHA256SUMS的文件中的值，完全一致即可。如果不一致，请重新下载Debian Live iso映像文件。

驿窗注：SHA256哈希值

SHA256SUMS文件中有多个SHA256哈希值，它们分别对应不同的iso映像文件，找到您下载的映像文件所对应的哈希值，然后对比即可。

哈希值不一致时，表示下载到本地的文件已经损坏，请务必重新下载，并重新校验，直到哈希值一致为止。

驿窗注：Linux系统中的检验方法

如何您是使用Linux系统进行SHA256校验，比如在Debian系统中，可以同时下载iso文件和SHA256SUMS文件，下载好以后把这两个文件放到同一目录中，注意文件名不要修改，然后打开终端，切换到此目录，运行下面的命令：

sha256sum -c SHA256SUMS

输出的内容应该类似下面这样：

debian-12.4.0-amd64-DVD-1.iso: 成功
debian-12.4.0-amd64-DVD-1.iso.torrent: 成功
sha256sum: debian-12.4.0-amd64-DVD-10.iso: 没有那个文件或目录
debian-12.4.0-amd64-DVD-10.iso: 打开或读取失败
sha256sum: debian-12.4.0-amd64-DVD-11.iso: 没有那个文件或目录
debian-12.4.0-amd64-DVD-11.iso: 打开或读取失败
……
……

第一行和第二行最右面的成功字样表示文件debian-12.4.0-amd64-DVD-1.iso和文件debian-12.4.0-amd64-DVD-1.iso.torrent已经下载成功，没有损坏，可以放心使用。

其下面的没有那个文件或目录表示命令没有找到对应的左侧的文件。

4. 下载U盘创建工具：Win32 Disk Imager

Win32 Disk Imager是Debian官方推荐的启动U盘制作工具，下载地址在这里： https://sourceforge.net/projects/win32diskimager/

下载并安装完成后，就可以开始创建启动U盘了。

5. 创建启动U盘

创建启动U盘，其实就是用Win32 Disk Imager把Debian Live iso映像写入到U盘中，方法如下：

第一步，把准备好的U盘插到电脑上；

第二步，打开Win32 Disk Imager：

Win32 Disk Imager

第三步，点击映像文件框右侧的文件夹图标，选择iso映像文件：

选择映像文件

这里请注意，右下角默认文件类型是.img .IMG，需要您点击这里然后选择*.*，之后才可以选择下载的iso映像文件。

第四步，设备处选择U盘，然后点击下面的“写入”按钮：

选择设备后点击写入按钮

注意设备不要选错。

点击写入按钮后等待完成即可。完成后，就可以用这个U盘引导电脑来体验Linux系统了。

6. 用启动U盘引导您的windows电脑

启动U盘已经创建完成，接下来就可以用这个U盘来启动电脑了。

电脑启动时，可以从硬盘启动，也可以从U盘启动。但到底先从哪个启动，需要在电脑的bios中设置，大部分电脑的主板，都可以在启动时不停地按Del键进入bios菜单，然后找到启动，下面有boot option #1，这个设置成U盘即可。(把启动U盘插到电脑上后再设置，否则bios检测不到U盘)

如果您找不到在哪里设置，可以尝试联系电脑主板制造商的售后服务咨询，他们通常都很愿意解答此类问题；或者，有朋友比较会电脑的，也可以问一部。

从启动U盘引导电脑后，应该可以看到下面的界面：

U盘启动电脑后的第一个操作界面

这里保持选择第一项即可，直接按回车键，就可以启动Debian系统。

7. 体验Debian

体验过程虽然只是随意看看，但下面内容也许对您有所帮助：

  1. 如果后续希望在自己的电脑上安装Debian，那么请务必测试下面三个内容：

  1. 无线网络和有线网络是否可以正常使用；

  2. 检查当前分辨率是否与windows中的分辨率相同；

  3. 尝试通过网络播放视频；

  4. (上面三项内容有任何一项不正常，表示您需要向其他人求助，否则可能无法成功安装Debian)

  1. 不要尝试修改本电脑的硬盘数据，以防误操作造成数据损失；

  2. 有些功能需要您对Debian有进一步了解才能尝试，比如从官方仓库安装软件，具体方法请参考本书后面的章节；

  3. 不同的桌面环境，操作有可能不同，例如xfce桌面环境中修改分辨率，在桌面上点击右键无效，需要点击 应用程序→设置→显示 才可以修改分辨率；

  4. windows系统的很多功能在Debian上的实现可能是完全不同的操作过程，所以，如果没有找到某个windows功能的时候，不要担心，可以到本书后面的三个“对应的windows功能”的章节查看，gnome、kde、xfce三种桌面都有说明；

  5. 输入法可能没有正确配置(配置方法请参考本书后面章节)；

  6. 有任何问题可以尝试到以下Linux论坛求助：

  1. Arch Linux 中文论坛

  2. Debian 中文论坛

  3. OpenSUSE 中文论坛
