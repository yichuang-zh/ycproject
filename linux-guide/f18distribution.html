了解一下那些发行版

下面列出的这些发行版都具有自己的特色，可以稍作了解。

一、 Arch Linux

  1. wiki文档很丰富，汉化比例比较高。入门学习其它Linux发行版时，经常也会去看Arch的wiki文档，因为文档写得很详细也很全面，尤其英文版本。

  2. 安装起来比Debian复杂很多(比如安装无线网卡驱动或者GUI)，有时候感觉像是在搭积木，一点一点把系统搭建起来。新手如果没有在虚拟机里练习过，建议不要在真机上尝试安装，因为大概率会安装失败。

  3. 如果能够熟练安装和使用甚至能基本维护Arch Linux，那么其它发行版上手基本都不是事儿。驿窗认为，Arch Linux是通往Linux真正入门的必经之路。

  4. DIY，高度可定制，滚动式发布。

  5. 中文用户可以到Arch Linux中文论坛可以交流。

二、 Debian

  1. 当前世界上有大概280个Linux发行版，其中近一半是基于Debian的，比如Ubuntu。这完全是市场行为的结果，非人为干预。(数据未经验证)

  2. 国产操作系统，主要是基于Debian系和RedHat系。

  3. 支持的硬件架构相对来说最全面。

  4. 是纯粹的社区发行版，不受商业行为规则的束缚，仅需要处理许可证问题。

  5. 仓库提供的应用种类相对更多更全。

  6. 安装使用相对更简单。

  7. 中文用户可以到Debian中文论坛及Ubuntu中文论坛交流。

三、 Gentoo Linux

  1. 可以把Gentoo Linux简单理解为Arch Linux的进阶版本：比Arch Linux多了一个编译的步骤，其它方面与Arch Linux在一定程度上相似。

  2. 能自己熟练安装Gentoo Linux的人，对于Linux的理解更深刻。

  3. 对于有时间有精力并且还有兴趣的用户，推荐尝试。

  4. 家用不推荐，很浪费时间和精力。

  5. DIY，高度可定制，滚动式发布。

四、 Kali Linux

  1. 基于Debian。

  2. 专门用于网络安全审计，即检查网络与系统的安全漏洞。

  3. 集成大量安全相关的工具，供测试人员使用。

  4. 非家用或工作站用。

五、 Manjaro

  1. 基于Arch Linux。

  2. 简单理解的话，可以看作是Arch Linux的傻瓜版本：安装过程比Arch Linux容易很多，其它方面与Arch Linux基本一致。

  3. 新手不推荐使用，建议先熟悉Arch Linux。

  4. 中文用户可以到Arch Linux中文论坛交流。

六、 openSUSE

  1. 来自德国，欧洲使用人群相对较多。

  2. 同时提供滚动式发布版本和版本式发布版本。

  3. 背后是商业发行版本SUSE Linux Enterprise，此版本有商业支持。openSUSE是社区版本，由SUSE提供支持。

  4. 基于RPM包，RedHat同样基于RPM包。

七、 Puppy Linux

  1. 主要特点是小，即Mini Linux。

  2. 类似的Mini Linux，还有诸如Tiny Core Linux、SLAX、SliTaz等。

  3. 不适合作为家用桌面系统使用。

八、 Qubes OS

  1. 基于Fedora。

  2. 专为保护隐私和安全而设计的Linux发行版。

  3. 号称即使已经被恶意黑客盯上，仍能保证系统安全。

  4. 由于广泛使用Xen虚拟化技术，所以需要更多内存。

  5. 强烈建议兴趣者阅读官网的硬件要求与硬件支持列表。

  6. 包含U盘防护功能。

  7. 等级式安全设计。

  8. 不适合虚拟机使用。

  9. Edward Snowden推荐。

  10. 可融合Whonix。

九、 RHEL

  1. RedHat公司提供的商业支持版本，专为服务器使用。

  2. 不适合家用。

  3. 在财力允许的情况下，RHEL或者SUSE Linux Enterprise是比较好的商业支持发行版选择。

  4. RedHat对于Linux的基础应用贡献无可比拟。

  5. RedHat被IBM收购以后，虽然无法预测未来走向，但如果被IBM以外的其它公司收购，结果大概率会更糟糕。

十、 SystemRescueCD

  1. 基于Gentoo。

  2. 可制作成可启动光盘/U盘。

  3. 专门用于修复损坏的系统及拯救系统中的数据。

  4. 集成多种相关的系统工具。

十一、 Tails

  1. 基于Debian。

  2. 专门用于防止隐私泄露及安全防护。

  3. Edward Snowden推荐。

  4. 系统关闭会导致所有相关信息被抹除，不留痕迹：闭后即焚。

  5. 可以设置关机后仍保存的内容。

  6. 使用luks和OpenPGP加密技术。

  7. 使用nautilus-wipe删除技术。

  8. 适合虚拟机使用或者启动U盘使用。

  9. 另一个相类似的发行版是基于Ubuntu的Kodachi。

十二、 Ubuntu

  1. 基于Debian。

  2. Ubuntu分为desktop和server两个方向。

  3. 非社区驱动；可购买商业支持。

  4. 资金无法支持RedHat和openSUSE的小企业，可能会选择Ubuntu。

  5. 实质上的技术含量无法与RedHat相比，不是一个量级。

  6. 国外个人用户量大。

  7. 云虚拟主机使用量大。

  8. 本质上是雇佣Debian开发者做了个Debian的变体？(这个中国人很擅长？)

  9. 很多发行版是基于Ubuntu的。

十三、 Whonix

  1. 基于Debian。

  2. 专注于隐私和在线安全防护。

  3. 运行时需要同时运行两个虚拟机，一个是工作站，用来操作；另一个是网关，用来防护。

  4. 从形式上看，其安全在线防护高于Tails。

  5. 从技术上看，隐私泄露的防护上可能不如Tails。

  6. 适合虚拟机使用。

  7. 可被Qubes OS融合。

十四、 (嵌入式，比如路由器系统OpenWrt)

  1. 开源路由器操作系统，由Linux内核发展而来。

  2. 插件丰富。

  3. 可按需定制自己的路由器。

  4. 新手有一定难度。

十五、 (FreeBSD)

  1. BSD不是Linux。

  2. BSD与Linux同源，都是源于UNIX。

  3. 稳健和安全是其特性。

  4. 硬件支持不如Linux。

  5. 不适合家用操作系统。

  6. 著名软件防火墙pfSense基于FreeBSD。

  7. BSD还有NetBSD和OpenBSD。

请注意

以上内容并未涵盖全部发行版，仅供了解Linux知识使用，不是发行版选择建议。

驿窗提示：

上面列出的这些Linux发行版是驿窗挑选出来的具有一定代表性的发行版，而导致它们能够成功的一个重要原因是Linux的开放性：百花齐放，百家争鸣。

所以，如果需要，您完全可以按自己意愿创建一个有专门用途或者特色的发行版。
